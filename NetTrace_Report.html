1. Project Overview

NetTrace is a four-layer graph-based fraud detection system built on the PaySim financial transaction dataset. Rather than scoring each transaction in isolation, NetTrace models every account as a node and every transaction as a directed edge in a network. This graph representation lets the system detect coordinated fraud patterns — such as chains of mule accounts rapidly passing stolen funds — that are invisible to row-by-row classifiers.

The system combines three complementary detection signals into a single ensemble risk score per account:

  • GraphSAGE (75%): A two-layer Graph Neural Network that learns fraud patterns from each account's own features and those of its network neighbours.
  • Louvain Community Detection (10%): Identifies tightly connected clusters of accounts and scores each cluster by its fraud concentration.
  • Isolation Forest (15%): Unsupervised anomaly detection that flags statistically unusual behavioural patterns without requiring fraud labels.

Accounts scoring above a configurable threshold (default 0.60) are flagged as high-risk. The system is designed to prioritise precision — minimising false alarms — making its output actionable for human investigators.

4. Node Feature Table

Each account node in the graph is described by seven features computed from its transaction history:

Feature Plain-English Description Why It Signals Fraud
in_degree Number of accounts sending money into this account Mule accounts often receive from many senders in a short window
out_degree Number of accounts this account sends money to Fraud originators tend to send to many destinations quickly
degree_ratio Ratio of outgoing connections to incoming connections High ratio (>1) means the account sends far more than it receives — a classic fraud pattern
avg_balance_drop Average fraction of account balance drained per transaction, clipped to [0, 1] Fraudsters typically drain accounts completely; legitimate senders rarely send 100% of balance
avg_amount_to_balance Average remaining balance fraction after each transaction (newBalance / oldBalance) Values near zero indicate the account is regularly emptied — a strong fraud signal
dest_empty_rate Fraction of transactions sent to accounts with zero opening balance Fraudsters frequently send to freshly created empty accounts to obscure trails
avg_dest_increase Average change in the destination account's balance after receiving a transaction Unusually large destination balance increases can indicate layering of stolen funds

5. Source Code

src/layer1_ingestion.py — Data loading, cleaning & transaction-level feature engineering
What it does: Loads the raw PaySim CSV and filters it down to the two transaction types where fraud actually occurs: TRANSFER and CASH_OUT. It then engineers four transaction-level features using vectorised NumPy operations — avoiding slow row-wise apply() calls. balance_drop_ratio measures how much of a sender's balance was sent; amount_to_balance_ratio measures how much balance remains after the transfer; dest_was_empty flags whether the destination account had zero balance before receiving; and dest_balance_increase tracks the raw balance change at the destination. These four signals feed into the node-level feature aggregation in Layer 3. 
# layer1_ingestion.py

import pandas as pd
import numpy as np
import os

FILEPATH = os.path.join(os.path.dirname(os.path.abspath(__file__)), '..', 'data', 'transactions.csv')

    df = pd.read_csv(filepath)
    print(f"Loaded {len(df):,} transactions")
    return df


def clean_data(df):
    cols = ['step', 'type', 'amount', 'nameOrig', 'nameDest',
            'oldbalanceOrg', 'newbalanceOrig',
            'oldbalanceDest', 'newbalanceDest', 'isFraud']
    df = df[cols].copy()
    df = df[df['amount'] > 0]
    df = df[df['type'].isin(['TRANSFER', 'CASH_OUT'])]
    print(f"Cleaned: {len(df):,} transactions remaining")
    return df


def engineer_features(df):
    df['balance_drop_ratio'] = np.where(
        df['oldbalanceOrg'] > 0,
        df['amount'] / df['oldbalanceOrg'], 0
    ).clip(0, 1)
    df['dest_balance_increase'] = df['newbalanceDest'] - df['oldbalanceDest']
    df['amount_to_balance_ratio'] = np.where(
        df['oldbalanceOrg'] > 0,
        df['newbalanceOrig'] / df['oldbalanceOrg'], 0
    ).clip(0, 1)
    df['dest_was_empty'] = (df['oldbalanceDest'] == 0).astype(int)
    return df
src/layer2_graph.py — Stratified sampling & directed graph construction
What it does: Converts the flat transaction table into a directed graph using NetworkX. Each unique account becomes a node; each transaction becomes a directed edge weighted by amount. The stratified sampler inflates fraud to 20% of the working sample — necessary because PaySim's raw fraud rate is under 0.1%, which would make training impractical. A critical design decision is that both the sender and the receiver of a fraudulent transaction are labelled as fraud nodes. PaySim's ground truth only labels the sender, but the receiver is a mule account that is equally part of the fraud network. This bilateral labelling is what allows GraphSAGE to learn from the full fraud subgraph rather than just half of it. 
# layer2_graph.py

import pandas as pd
import networkx as nx

def stratified_sample(df, fraud_ratio=0.2, max_size=50000):
    """Inflate fraud to 20% so the GNN has enough signal to learn from."""
    fraud   = df[df['isFraud'] == 1]
    clean   = df[df['isFraud'] == 0]
    n_fraud = len(fraud)
    n_clean = int(n_fraud * ((1 - fraud_ratio) / fraud_ratio))
    total   = n_fraud + n_clean
    if total > max_size:
        scale   = max_size / total
        n_fraud = int(n_fraud * scale)
        n_clean = int(n_clean * scale)
    return pd.concat([
        fraud.sample(n=n_fraud, random_state=42),
        clean.sample(n=n_clean, random_state=42)
    ]).reset_index(drop=True)


def build_graph(df):
    """Build directed graph; label both sender AND receiver as fraud."""
    G = nx.DiGraph()
    for _, row in df.iterrows():
        sender, receiver = row['nameOrig'], row['nameDest']
        is_fraud = int(row['isFraud'])
        if G.has_edge(sender, receiver):
            G[sender][receiver]['weight'] += row['amount']
        else:
            G.add_edge(sender, receiver, weight=row['amount'])
        # Label both accounts — receiver is a mule, not just the sender
        for acct in [sender, receiver]:
            if acct not in G.nodes:
                G.add_node(acct, is_fraud=is_fraud)
            else:
                G.nodes[acct]['is_fraud'] = max(
                    G.nodes[acct].get('is_fraud', 0), is_fraud
                )
    return G
src/utils.py — Shared model architecture, feature engineering & detection utilities
What it does: Houses all shared logic used by both the detection pipeline (layer3_detection.py) and the export runner (export_results.py). Contains five components: (1) the GraphSAGE model class — a two-layer graph neural network with ReLU activation and 30% dropout; (2) engineer_node_features() — aggregates transaction-level features per account into node-level statistics; (3) build_pyg_data() — converts the NetworkX graph into a PyTorch Geometric Data object with a stratified 80/20 train/test split; (4) run_clustering() — applies Louvain community detection and scores each cluster by fraud concentration; (5) run_isolation_forest() — fits an Isolation Forest with contamination=0.05 to flag statistically anomalous accounts.

class GraphSAGE(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels):
        super().__init__()
        self.conv1 = SAGEConv(in_channels, hidden_channels)
        self.conv2 = SAGEConv(hidden_channels, out_channels)

    def forward(self, x, edge_index):
        x = self.conv1(x, edge_index)
        x = F.relu(x)
        x = F.dropout(x, p=0.3, training=self.training)
        x = self.conv2(x, edge_index)
        return F.log_softmax(x, dim=1)


def engineer_node_features(G, df):
    account_stats = {}
    for _, row in df.iterrows():
        for acct in [row['nameOrig'], row['nameDest']]:
            if acct not in account_stats:
                account_stats[acct] = {
                    'balance_drops': [], 'amount_to_balance': [],
                    'dest_empty': [], 'dest_increases': []
                }
        s = row['nameOrig']
        account_stats[s]['balance_drops'].append(row['balance_drop_ratio'])
        account_stats[s]['amount_to_balance'].append(row['amount_to_balance_ratio'])
        account_stats[s]['dest_empty'].append(row['dest_was_empty'])
        account_stats[s]['dest_increases'].append(row['dest_balance_increase'])
    rows = []
    seen = set()
    for node in G.nodes():
        if node in seen: continue
        seen.add(node)
        in_deg  = G.in_degree(node)
        out_deg = G.out_degree(node)
        stats   = account_stats.get(node)
        degree_ratio = out_deg / in_deg if in_deg > 0 else float(out_deg)
        rows.append({
            'node': node, 'in_degree': in_deg, 'out_degree': out_deg,
            'degree_ratio': round(degree_ratio, 4),
            # ... other feature averages from account_stats ...
            'is_fraud': G.nodes[node].get('is_fraud', 0)
        })
    return pd.DataFrame(rows)


def run_clustering(G, features_df):
    partition = best_partition(G.to_undirected(), randomize=False)
    features_df = features_df.copy()
    features_df['community'] = features_df['node'].map(partition)
    cs = (features_df.groupby('community')['is_fraud']
          .agg(['mean', 'count']).reset_index())
    cs.columns = ['community', 'fraud_rate', 'size']
    cs = cs[cs['size'] >= 5].sort_values('fraud_rate', ascending=False)
    return features_df, cs

src/export_results.py — Full pipeline runner: trains, scores, and exports results.pkl
What it does: The main pipeline entry point. Orchestrates all four layers: loads and cleans data, builds the graph, engineers node features, runs all three detection models, computes the ensemble score, and serialises everything to data/results.pkl for the dashboard to consume. Key design choices include: early stopping with patience=30 and a minimum improvement delta of 0.0001 to avoid stopping on AUC fluctuations; a decision threshold of 0.60 (raised from 0.50) to improve precision; and ensemble weights of GraphSAGE 75%, Community 10%, Isolation Forest 15%.

                   run_clustering, run_isolation_forest)


def compute_ensemble_score(features_df):
    """Combines GNN + community + anomaly into one per-account score."""
    features_df['ensemble_score'] = (
        0.60 * features_df['graphsage_probability'] +
        0.25 * features_df['community_signal']      +
        0.15 * features_df['anomaly_signal']
    )
    threshold = 0.5
    flagged   = features_df[features_df['ensemble_score'] > threshold]
    confirmed = flagged[flagged['is_fraud'] == 1]
    precision = len(confirmed) / len(flagged)
    recall    = len(confirmed) / features_df['is_fraud'].sum()
    print(f"Precision: {precision:.1%}    Recall: {recall:.1%}")
    return features_df.sort_values('ensemble_score', ascending=False)
